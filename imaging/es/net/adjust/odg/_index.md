{{% blocks/products/pf/agp/content h2="Cómo ajustar archivos ODG usando C#" %}}

Incluso los fotógrafos más experimentados pueden cometer errores o encontrar condiciones de iluminación desfavorables durante una sesión. Estas situaciones pueden provocar defectos en la imagen, pero incluso una mala fotografía tiene posibilidades. Con la biblioteca C#, puede corregir estos problemas mediante programación ajustando el brillo, el contraste y la gamma de color. Si una foto resulta demasiado oscura, aumentar el brillo aclarará las áreas oscuras y hará visibles los detalles previamente ocultos. Ajustar el contraste aumenta la diferencia entre las áreas claras y oscuras, ampliando el rango de brillo para una imagen mejorada. Si ha capturado una fotografía con iluminación artificial y el resultado es un tono de color no deseado, utilice el ajuste de gamma de color para corregir el balance de blancos. Para ajustar archivos ODG, usaremos [Aspose.Imaging para .NET](https://products.aspose.com/imaging/net) API, que es una API de conversión y manipulación de imágenes rica en funciones, potente y fácil de usar para la plataforma C#.     {{% blocks/products/pf/agp/content h2="Acerca de Aspose.Imaging para .NET API" %}}


Aspose.Imaging API es una solución de procesamiento de imágenes para crear, modificar, dibujar o convertir imágenes (fotos) dentro de las aplicaciones. Ofrece: procesamiento de imágenes multiplataforma, que incluye, entre otros, conversiones entre varios formatos de imagen (incluido el procesamiento uniforme de imágenes de varias páginas o varios cuadros), modificaciones como dibujar, trabajar con primitivas gráficas, transformaciones (redimensionar, recortar, voltear y rotar , binarización, escala de grises, ajuste), funciones avanzadas de manipulación de imágenes (filtrado, difuminado, enmascaramiento, corrección del sesgo) y estrategias de optimización de la memoria. Es una biblioteca independiente y no depende de ningún software para las operaciones de imagen. Uno puede agregar fácilmente funciones de conversión de imágenes de alto rendimiento con API nativas dentro de los proyectos. Estas son API locales 100 % privadas y las imágenes se procesan en sus servidores.
