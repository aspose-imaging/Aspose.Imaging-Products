Винахід кольорової плівки став важливою віхою у сфері фотографії. Однак класична фотографія невід’ємно пов’язана з чорно-білими зображеннями. Незважаючи на широкі технічні можливості камер для захоплення повного спектру кольорів, багато людей вибирають монохромний режим, перетворюючи свої знімки на чорно-білі. У таких випадках у дію вступає функція бінаризації, яка перетворює всі пікселі на двійкові значення: \"0\" для білого і \"1\" для чорного. В інших сценаріях це перетворення обумовлено не художніми рішеннями, а практичними, як-от підготовка чорно-білих ілюстрацій для друку в книгах або газетах. Використовуючи графічну бібліотеку Java, ви можете призначити поріг яскравості пікселя. Пікселі зі значеннями яскравості нижче цього порогу стануть чорними, тоді як пікселі вище цього порогу стануть білими. Крім того, ви можете застосувати адаптивний метод бінаризації, який враховує значення пікселів у навколишній області. Цей підхід призводить до більш плавних переходів між межами кольорів на чорно-білому зображенні, що в результаті виходить. Для того, щоб двійковий файл J2K, ми будемо використовувати [Aspose.Imaging for Java](https://products.aspose.com/imaging/java) API, який є багатофункціональним, потужним і простим у використанні API для обробки зображень і перетворення для платформи Java. Ви можете завантажити його останню версію безпосередньо з [Maven](https://repository.aspose.com/webapp/#/artifacts/browse/tree/General/repo/com/aspose/aspose-imaging) і встановити на своєму Maven на основі проекту, додавши такі конфігурації до pom.xml.

Він пропонує: кросплатформну обробку зображень, включаючи, але не обмежуючись, перетворення між різними форматами зображень (включно з уніфікованою обробкою багатосторінкових або багатокадрових зображень), такі модифікації, як малювання, робота з графічними примітивами, перетворення (зміна розміру, обрізання, перевертання та обертання). , бінаризація, відтінки сірого, коригування), розширені функції обробки зображення (фільтрування, згладжування, маскування, виправлення) і стратегії оптимізації пам’яті. Це окрема бібліотека, яка не залежить від програмного забезпечення для роботи із зображеннями. Можна легко додати високоефективні функції перетворення зображень за допомогою власних API у проекти. Це 100% приватні локальні API, а зображення обробляються на ваших серверах.
